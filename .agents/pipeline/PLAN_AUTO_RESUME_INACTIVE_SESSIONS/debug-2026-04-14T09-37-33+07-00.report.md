## Chi tiết nguyên nhân

### Root cause #1 — resume path bỏ mất `yolo`

**Evidence 1 — auto-resume spawn không truyền `yolo`:**

File: `hub/src/sync/syncEngine.ts:492-503`

```ts
const spawnResult = await this.rpcGateway.spawnSession(
    targetMachine.id,
    metadata.path,
    flavor,
    session.model ?? undefined,
    undefined,
    undefined,
    undefined,
    undefined,
    resumeToken,
    session.effort ?? undefined
)
```

- Tham số thứ 6 của `spawnSession(...)` là `yolo`.
- Ở đây giá trị truyền vào là `undefined`.
- Kết quả: runner không biết session cũ từng được tạo ở YOLO mode.

**Evidence 2 — runner chỉ add `--yolo` khi nhận bool `true`:**

File: `cli/src/runner/run.ts:197-200,352-374`

```ts

...

  args.push('--yolo');
}
```

- Nếu resume path truyền `undefined`, biến `yolo` sẽ là `false`.
- Khi đó process Claude resumed sẽ không có `--yolo`.

**Evidence 3 — Claude chỉ bypass approval khi có `--yolo`:**

File: `cli/src/commands/claude.ts:39-44`

```ts
} else if (arg === '--yolo') {
    options.permissionMode = 'bypassPermissions'

} else if (arg === '--dangerously-skip-permissions') {
    options.permissionMode = 'bypassPermissions'
    unknownArgs.push(arg)

```

- `--yolo` là điểm kích hoạt `bypassPermissions` cho Claude.
- Không có flag này thì Claude resume ở mode mặc định.

## Đề xuất fix

### Fix chính

Ưu tiên 1:
- Trong `hub/src/sync/syncEngine.ts:492-503`, derive `yolo` từ session permission mode đã lưu.
- Với Claude, nếu `session.permissionMode === 'bypassPermissions'` thì truyền `true` vào tham số `yolo` của `rpcGateway.spawnSession(...)`.
- Với các flavor khác, map tương ứng nếu permission mode là `yolo`.

Ví dụ hướng fix:
- Claude: `const resumeYolo = session.permissionMode === 'bypassPermissions'`
- Codex/Gemini/Cursor/OpenCode: map theo mode enum của từng flavor (`'yolo'`, `'safe-yolo'`, ...)

Ưu tiên 2 (safety net):
- Sau khi `waitForSessionActive(...)` thành công, gọi `applySessionConfig(..., { permissionMode: session.permissionMode, model: session.model, effort: session.effort })` cho resumed session.
- Điều này giảm mismatch giữa state hub và runtime process.
